## Repository Structure
Each practical resides in its own self-contained folder, ensuring that code, documentation, and output evidence are never mixed across assignments. This structure supports:
* Independent evaluation of each practical
* Clean version history per assignment
* Easy scalability as new practicals are added weekly

### Folder Architecture
```text
Problem-Solving-with-Python/
│
├── Practical_01/
│   ├── README.md            # Aim, logic, and explanation
│   ├── practical_01.py      # Python source code
│   └── output.png           # Execution screenshot
│
├── Practical_02/
│   ├── README.md
│   ├── practical_02.py
│   └── output.png
│
├── Practical_XX/
│   ├── README.md
│   ├── practical_xx.py
│   └── output.png
│
└── README.md                # Repository index (this file)

> Each individual practical `README.md` contains the problem statement, the approach or algorithm used, and a brief explanation of the solution logic.

## Documentation Standards

To maintain consistency and readability across all practicals, each folder follows a uniform documentation format:

- **Aim** — A concise statement of the practical's objective
- **Code** — Clean, commented, and readable Python source
- **Output** — A screenshot confirming successful execution
- **Explanation** — A short summary of the logic or concept applied

This consistency ensures that every practical can be reviewed independently without loss of context.

## How to Run

1. Clone the repository:
```bash
   git clone <repository-url>
```
2. Navigate to the desired practical folder:
```bash
   cd Practical_01
```
3. Run the Python script:
```bash
   python practical_01.py
```
